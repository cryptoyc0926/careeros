# 简历排版与结构规则（Layout Rules）

> **适用范围**：CareerOS 所有简历输出场景（PDF / Word / HTML 预览）
> **规则版本**：v1.0（2026-04-20）
> **配套文件**：
> - `resume_bold_rules.md` — 加粗规则
> - `resume_system.md` — AI 生成内容规则（DEAI + 加粗精简版）
> - `resume_prompt_rules.py` — 规则常量（代码层引用）
> **维护原则**：布局规则改动必须同步本文件 + Obsidian `04_CAREEROS/简历布局规则.md`

---

## 零、最高原则

1. **一页 A4，不溢出**。任何优化都要先保证这一条。
2. **HR 30 秒扫读**：头部 3 秒 → 个人总结 7 秒 → 项目经历 15 秒 → 其他 5 秒。排版服从这个时间预算。
3. **信号密度 > 信息完整度**。宁可删 2 条 bullet 留白，不要塞满 5 条。

---

## 一、页面与排版基线

| 维度 | 值 | 备注 |
|---|---|---|
| 纸张 | A4（21.0 × 29.7 cm） | 不用 Letter |
| 页边距 | 上 1.0cm / 下 0.8cm / 左右 1.5cm | 下比上窄，视觉重心上移 |
| 正文字号 | 9.5pt | 不小于 9pt，不大于 10pt |
| 行高 | 1.4–1.5 | 紧凑但不挤 |
| Section 标题字号 | 12pt | 黑体 + 灰底 `#e7e6e6` |
| 姓名字号 | 32–36pt | 楷体（Kaiti SC / STKaiti / KaiTi），字间距 12–14pt |
| 联系方式 / 意向 字号 | 10pt | 用 `\|` 或 ` | ` 分隔 |

---

## 二、头部区（必须元素）

### 2.1 左右两列布局

```
左列（宽 14cm）              右列（宽 3cm）
────────────────────         ─────────────
姓名（居中楷体）              证件照
电话 | 邮箱                   68×90pt
求职意向 | 城市 | 到岗         （约 2.4cm 宽）
```

### 2.2 照片规范

- 尺寸：**68pt × 90pt**（约 2:3 证件照比例）
- HTML：`object-fit: cover; object-position: center top;`（面部居中偏上，肩膀裁掉多余部分）
- Word：用无边框 2 列表格实现右对齐；`add_picture(..., width=Cm(2.4))`
- 放不进：保留 `photo` 灰色虚线框占位符（`onerror` 回退）

### 2.3 联系方式

- **只放电话 + 邮箱两项**
- **不放**：X / GitHub / LinkedIn / Twitter 等社媒账号（这些放到项目经历里作为作品链接）
- **不放**：地址、性别、年龄、民族、政治面貌

### 2.4 求职意向

- **必须是目标 JD 的原岗位名**（或最接近原岗位名的表述）
  - ✅ "国内社媒 KOL 增长运营实习"（Kimi JD 原名）
  - ✅ "AI 产品运营实习"（同花顺 JD 原名）
  - ❌ "AI 管培生"（笼统）/ "增长运营"（太泛化）
- 到岗时间：备注清楚（例：一周内 / 下周到岗 / 每周可实习 5 天）

## 三、Section 结构（5 大块，固定顺序）

**标准顺序**：个人总结 → 项目经历 → 实习经历 → 专业技能 → 教育背景

| 是否可调整 | 场景 |
|---|---|
| 🔒 固定 | 应届/实习生投递，项目经验是主卖点 |
| 🔄 可调 | 有 3 年+ 正式工作经验时，把「工作经历」上移到「项目经历」之前 |

### 3.1 section 标题样式

```css
.section-title {
    background: #e7e6e6;
    padding: 2pt 6pt;
    font-family: "SimHei", "Heiti SC", "Microsoft YaHei";
    font-size: 12pt;
    font-weight: 700;
    margin-top: 5pt;
    margin-bottom: 2pt;
}
```

统一用灰底 `#e7e6e6` 块状条，不用下划线或边框。

## 四、个人总结（70–140 字）

### 4.1 结构模板

> **[身份定位]**＋**[硬核数据]**＋**[工具栈]**＋**[实习覆盖面]**＋**[软素质]**

示例（同花顺版）：
> 在校期间独立运营 AI × 金融信息方向内容账号 X @Cady_btc，零投放做到 **9,000+** 粉、Telegram 社群 **1,300+** 订阅、完成 **20+** 次品牌商单，熟悉海内外社交平台内容生态。日常用 Claude Code、Kimi、ChatGPT、Cursor 做内容生产与小工具开发。两段运营实习覆盖海外 AIGC 冷启动（TikTok / Instagram / Reddit）、竞品拆解、活动数据复盘与跨部门协作。英语 CET-6，可独立完成英文内容撰写。应用统计出身，习惯用数据面板复盘每条内容的真实表现。

### 4.2 写作规则

- 开头第一句必须带**核心数据**（粉丝数、商单数、增长率等）
- 中间列出**AI 工具栈**（3–4 个主力工具，按 JD 关键词排序）
- 结尾**软素质**用自然句式（避免 "具备良好的学习能力" 这类模板话）
- 禁用路标词：更关键 / 事实上 / 值得注意 / 总之 / 换句话说

## 五、项目 / 实习经历（正文主体）

### 5.1 三段式 header

```
[公司/项目名(左)]  [职位/角色(居中)]  [日期(右)]
```

- 三列均加粗
- 用表格实现（不用 flexbox，Word/PDF 兼容性最稳）
- 公司名用原文（不要缩写），括号可加 "（开源）""（实习）" 等标注

### 5.2 Bullet 结构

```
• **label：** 正文（含粗体纯数字）
```

- **label 必须加粗**，冒号全角 `：`
- label 简洁：4–8 个字（例：内容生产线、自然流量破圈、活动数据复盘）
- 正文 50–90 字/条，不超过 2 行
- 每个 item 的 bullet 数量：**2–4 条**（<2 说明内容不够厚，>4 说明该拆项或精简）

### 5.3 开源/作品项目的链接规范

开源项目或个人作品的最后一条 bullet：
```
• **网站链接：** xxx　｜　**GitHub：** xxx
```

- 两个链接中间用全角 `｜` 分隔（视觉呼吸）
- URL 去掉 `https://` 前缀（视觉干净）
- 链接**纯黑无下划线**（见第八节）

### 5.4 动词优先级

| 优选 | 禁用 |
|---|---|
| 搭 / 写 / 跑 / 做 / 拆 / 带 / 调 | 赋能 / 驱动 / 构建 / 打造 |
| 主导 / 独立完成 / 交付 | 助力 / 深耕 / 聚焦 / 洞察 |

## 六、专业技能（最多 3 行）

### 6.1 标准三分法

```
• **AI 工具与内容生产：** [AI 工具清单]；[设计/生产工具]
• **海内外社媒：** [深度运营平台]；[深度使用平台]    ← 只有当该岗位强调社媒时才保留
• **数据、开发与语言：** [技术栈]；英语 CET-x、计算机 x 级
```

**硬约束**：
- 超过 3 行必须压缩（合并分类 / 删非核心工具）
- 不列 Word / Excel / PPT（默认技能，列了反而 low）
- 工具顺序按 JD 关键词优先级（Claude Code / Kimi / ChatGPT 等按目标公司的主要工具排首位）

### 6.2 常见取舍

| 场景 | 行为 |
|---|---|
| 该岗位强调海外社媒（如 Kimi KOL 岗） | 保留 "海内外社媒" 独立一行 |
| 该岗位不强调社媒（如同花顺 AI 产品运营） | 删掉 "海内外社媒" 行，并入工具行 |
| 页面放不下 | 先合并 "数据 + 开发 + 语言" 为一行 |

---

## 七、教育背景（1 行压缩）

**禁用三段式 header**（节省空间）。直接压成 1 条 bullet：

```
• **学校名** 专业·学历  日期  ｜  **核心课程：** xxx  ｜  **荣誉：** xxx
```

- 全角 `｜` 分隔
- 核心课程 4–5 门（选 JD 相关专业课）
- 荣誉简写（校级奖学金 ≠ 校三等奖学金也要保留，它证明你不垫底）

## 八、超链接处理（视觉去蓝）

### 8.1 HTML

```css
a, a:link, a:visited, a:hover, a:active {
    color: inherit;         /* 继承周围文字颜色（通常是黑色） */
    text-decoration: none;  /* 去下划线 */
}
```

### 8.2 效果

- 视觉上：邮箱、URL 都是**纯黑无下划线**
- 功能上：PDF 在 Preview / Adobe 里**仍可点击**（mailto: / https: 跳转）
- Word：python-docx 默认的 `add_run` 就是纯文本，无超链样式；如需加跳转可用 hyperlink 扩展，但通常**不必要**（应聘者填写链接的目的是让 HR 知道存在，能否点击次要）

### 8.3 哪些内容应该视觉去蓝

- ✅ 邮箱、个人网址、GitHub、作品 URL
- ✅ Telegram 频道链接
- ❌（**保留蓝色下划线**）：无此场景，简历里所有超链都按纯黑处理

## 九、压页策略（一页放不下时的取舍顺序）

从**影响最小**到**影响最大**，按顺序尝试：

1. **专业技能合并分类**（3 行 → 2 行）省 1 行
2. **教育背景去三段式 header**（改 1 条 bullet）省 1–2 行
3. **个人总结精简冗余描述**（删重复的平台/工具列举）省 1 行
4. **降低 line-height**（1.45 → 1.4）省 0.5 行/bullet
5. **删最弱的一条 bullet**（实习或项目里的数据最弱的那条）省 1 行
6. **section 上下 margin 压缩**（5pt → 3pt）省 1–2 行
7. **整体字号 9.5pt → 9pt**（最后手段，影响可读性）

**禁止做的**：
- 不要删整个项目或实习（经历完整性 > 排版美观）
- 不要删头像（视觉锚点很重要）
- 不要用 1 页以上的简历去投递实习岗

## 十、每次定制新 JD 的工作流

```
1. 读 JD → 提取 top 5 关键词
2. 改求职意向行 = JD 原岗位名
3. 改个人总结第一句 → 带上 JD 强调的数据
4. 微调项目/实习 bullet label → 用 JD 的措辞
5. 重排专业技能工具顺序 → JD 提到的工具排前
6. 重排 "海内外社媒" 行 → JD 不要求则删掉
7. 重排头部到岗时间备注 → JD 提到 "每周 X 天" 则加进去
8. 跑一次 HTML → Chrome headless → PDF
9. 跑一次 python-docx → Word
10. 人眼扫一遍：头像位置、数字加粗、超链颜色、一页容纳
```

## 十一、适用范围与例外

**强制适用**：
- 应届生 / 实习生投递（简历一页 A4）
- CareerOS 自动生成的定制版简历

**可放宽**：
- 3 年+ 工作经验（可以 2 页，但每页都要满）
- 海外英文简历（另立 `resume_layout_rules_en.md`）
- 学术简历（需独立规则）
