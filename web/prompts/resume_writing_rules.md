# 简历内容书写规则（Writing Rules）

> **适用范围**：CareerOS 所有简历内容生成场景（AI tailor / 人工改稿 / 新经历添加）
> **规则版本**：v1.0（2026-04-20）
> **配套文件**：
> - `resume_bold_rules.md` — 加粗规则（格式）
> - `resume_layout_rules.md` — 排版布局（格式）
> - `resume_system.md` — AI 生成 prompt（精简版）
> **维护原则**：写作规则改动必须同步 Obsidian `04_CAREEROS/简历写作规则.md`

---

## 零、最高原则

1. **具体 > 抽象**。凡是能写数字就写数字，能写公司/工具名就写公司/工具名。
2. **动词 > 形容词**。简历里不说"擅长"，直接写"做了什么"。
3. **结果 > 过程**。HR 关心"产生了什么"，不关心"你怎么想的"。
4. **真实 > 漂亮**。所有数据必须能当场验证（截图、账号、报表）。
5. **克制 > 堆砌**。宁可 3 条硬 bullet，不要 5 条水 bullet。

## 一、简历的语气与口吻

### 1.1 人称与主语

- **隐去第一人称**：不说"我负责"、"我参与"，直接用动词起头
  - ✅ "独立搭建 TikTok 官方账号"
  - ❌ "我独立搭建了 TikTok 官方账号"
- **禁用第二人称**：简历不是博客，不出现"你会"、"你可以"
- **主语省略时，上下文必须能补出谁在做**（通常是候选人本人）

### 1.2 时态与语气

- **中文简历默认现在完成时感**（用过去动作 + 现在结果）
  - "搭建了 SOP" ≠ "搭建过 SOP"，用"搭建"干脆
- **克制、数据化**，不用感叹号、反问句
- **不带情绪修饰**：禁"成功 / 完美 / 极大提升 / 显著增长"
  - ✅ "单场参与率提升 **18.9%**"
  - ❌ "显著提升了单场参与率，效果非常出色"

### 1.3 句式节奏

- bullet 里**短句 + 分句**，用逗号和分号串连
  - 避免超过 3 个分句叠罗汉
- 每个 bullet **不超过 2 行**（约 50–90 字）
- 列举内容用顿号「、」，**3 项为一组**（不多不少最舒服）

### 1.4 禁用词（DEAI · 参考 `resume_system.md` 第 11–15 条）

| 类别 | 禁用 | 改写方向 |
|---|---|---|
| 连接词模板 | 不是 A 而是 B / 一旦…就 / 通过…来… | 直接陈述事实 |
| AI 动词 | 赋能 / 驱动 / 构建 / 打造 / 助力 / 深耕 / 聚焦 / 拆解 | 用具体动词（搭 / 写 / 跑） |
| 比喻词 | 底色 / 信号 / 赛道 / 闭环 / 抓手 / 拼图 | 删除或直接写事实 |
| 降维引导 | 说白了 / 本质上 / 归根结底 / 敲黑板 | 简历里永远不需要引导 |
| 虚词模板 | 能够 / 具备 / 擅长 / 精通（独立出现时） | 改成具体案例 |

## 二、STAR 法则（简历专用变形）

### 2.1 原教旨 STAR vs 简历 STAR

| 原教旨 STAR | 简历变形 | 落在哪里 |
|---|---|---|
| Situation（背景） | 压进 header 或 label | 公司名 / 项目标题 / bullet label |
| Task（任务） | 和 S 合并 | 同上 |
| Action（动作） | bullet 前半段 | 正文前 50% |
| Result（结果） | bullet 后半段 + **粗体数字** | 正文后 50% |

**简历里不要写显式的 S/T/A/R 标签**——学生气太重。用隐式结构。

### 2.2 Bullet 的黄金结构

```
**[label：]** [动作动词 + 对象 + 方式]，[量化结果]
        ↑ S+T        ↑ A                      ↑ R
```

### 2.3 示例对照

**弱（只有 A，没有 R）**：
> 负责 TikTok 账号的内容运营和社区维护

**强（STAR 变形完整）**：
> **海外社媒 0 → 1：** 公司海外内容从零起步，独立搭建 TikTok + Instagram 官方账号，交付一套「AI 生产 → 二次编辑 → 多平台分发」SOP 文档；零投放条件下做到单条内容 **10,000+** 自然曝光，带动官网日均 UV 从个位数提升到 **200+**

解剖：
- `海外社媒 0 → 1：` = S+T 标签（背景：海外社媒从零开始）
- `公司海外内容从零起步，独立搭建 TikTok + Instagram 官方账号，交付一套...SOP` = A（动作）
- `零投放...10,000+...200+` = R（结果 + 量化）

## 三、拿到新经历 → 转成 bullet 的 6 步法

### Step 1: 回答 5 个问题

拿到一段经历（实习/项目/活动），先回答：

| 问题 | 答案示例 |
|---|---|
| **What** 做了什么？ | 搭了一个海外社媒矩阵 |
| **How** 怎么做的？ | TikTok + Instagram + 内容 SOP |
| **Scale** 规模 / 时长 / 投入？ | 3 个月，独立完成，零预算 |
| **Output** 产出什么交付物？ | 1 套 SOP 文档 + 2 个官方账号 |
| **Impact** 结果能证明什么？ | 单条 10k+ 自然曝光，官网 UV 200+ |

### Step 2: 筛选 2–3 条最硬的 Impact

一段经历通常能提炼 5–8 个事实，**只保留 2–4 个 bullet**。筛选标准：
- 有量化数字 → 优先
- 和目标 JD 关键词匹配 → 优先
- 是 0→1 或大幅提升 → 优先
- 是结果 > 过程 → 优先

### Step 3: 为每条 bullet 取 label（4–8 字）

Label 取名 4 种常见类型：

| 类型 | 示例 | 适用 |
|---|---|---|
| **职能型** | 内容生产线 / 活动数据复盘 | 描述工作职能 |
| **成果型** | 自然流量破圈 / 海外社媒 0 → 1 | 强调结果 |
| **方法型** | Prompt 工程 / 开发方法论 / 产品定义 | 强调方法论 |
| **对象型** | 达人共创 / 竞品调研 / 海外精准获客 | 强调做了什么对象 |

**label 命名原则**：
- ✅ 动宾或名词短语，简洁
- ✅ 用 JD 里出现过的措辞
- ❌ "负责"、"参与"、"协助"（虚词开头）
- ❌ "XX 工作"（"工作"是废话）

### Step 4: 写动作部分（bullet 前半）

用**动词开头 + 对象 + 方式**：
```
[动词] [对象] [方式/工具]
搭建    信息抓取管道   基于 X API + Claude + OpenCLaw
```

**动词选择表**：

| 场景 | 优选动词 | 禁用 |
|---|---|---|
| 从零搭建 | 搭 / 搭建 / 独立完成 / 跑通 | 构建 / 打造 |
| 改进优化 | 重做 / 改写 / 调优 / 升级 | 赋能 / 驱动 |
| 调研分析 | 拆 / 拆解 / 调研 / 挖 | 深耕 / 聚焦 / 洞察 |
| 对接协作 | 对接 / 带 / 配合 / 协同 / 主导 | 助力 |
| 输出交付 | 产出 / 交付 / 写 / 做出 | 打造 |

### Step 5: 写结果部分（bullet 后半）

**结果必须带数字**。组合方式：

| 组合 | 示例 |
|---|---|
| 绝对数 | 完成 **20+** 次品牌商单 |
| 相对数 | 参与率提升 **18.9%** |
| 绝对 + 相对 | 借款金额 **1,048 万 → 1,262 万（+20.6%）** |
| 时间维度 | **单场** 触达率从 **4%** 提升到 **10%** |
| 对比段 | **0 → 9,000+** 粉丝 |

**数字写作 5 条硬规则**（详见第六节）：
1. 原文数字字面保留（9,000 就是 9,000，不缩成 9k）
2. 带逗号（9,000 ✅ / 9000 ❌）
3. 时间维度加前缀（"单场 / 单月 / 单条"让数据更具体）
4. 对比段用箭头 → 
5. 百分号紧贴数字（不空格）

### Step 6: 合并、检查、压行

- 合并：A + R 中间用逗号或分号连，不写两句
- 检查：是否 50–90 字？超过就删枝叶
- 压行：能用顿号就不用"和"，能省副词就省（"非常"、"极大"、"成功"）

## 四、Bullet 写作模板与示例库

### 4.1 通用模板

```
**[label：]** [动词] [对象] [方式/工具/规模]，[量化结果 / 次级结果]
```

### 4.2 高频场景模板库

**场景 A：运营活动复盘**
```
**活动数据复盘：** 搭建活动日报 + 归因模型，**[绝对/相对数]**，参与人数 **[前 → 后]**，**[业务核心指标前 → 后（+%）]**
```

**场景 B：海外社媒冷启动**
```
**海外社媒 0 → 1：** 独立搭建 [平台 A] + [平台 B] 官方账号，交付一套 [SOP / 内容流程 / 分发链路] 文档；[零投放 / 限时]条件下做到 [单条 / 单月] **[XX,XXX+]** 自然曝光，带动 [官网 / 粉丝 / 付费] **[XX+]**
```

**场景 C：AI 工具 / Prompt 项目**
```
**[方法论 label]：** 基于 [工具栈 A + B + C] 搭建 [管道 / 流程]，每[日/周] [具体动作] **[XX+ 条素材/内容]**；[效率前 → 效率后，XX% 提升]
```

**场景 D：跨部门 / 跨团队协作**
```
**跨团队协作：** 主导 [岗位] 的 [SOP / 工具 / 模板] 升级，[Canva / Figma / Axure] [对象] 模板化，配合 [产品 / 运营 / 技术] 团队推进 [动作]，**[核心指标前 → 后]**
```

**场景 E：开源 / 个人作品**
```
**产品定义：** 独立拆出 **[X]** 个核心模块（[模块 1 / 2 / 3 / ...]），端到端覆盖 [业务域]；[量化规模，如"100+ XX"]
**技术实现：** [技术栈] 做 [架构层]，[关键设计点]；**[核心性能指标，如"30 秒产出"]**
**开发方法论：** 全流程 vibe coding，独立完成 [架构 / 开发 / 测试 / 文档 / 部署]；v[X.Y.Z] 已发布，含 [交付物清单]
**网站链接：** [域名]　｜　**GitHub：** [用户名/项目名]
```

## 五、个人总结 5 槽位模板

### 5.1 结构

个人总结 70–140 字，按以下 5 个槽位组装：

```
[槽位 1: 身份钩子]，[槽位 2: 核心数据]，[槽位 3: 工具栈]。
[槽位 4: 经历覆盖面]。[槽位 5: 能力锚点]。
```

| 槽位 | 字数 | 内容 | 示例（同花顺版） |
|---|---|---|---|
| 1. 身份钩子 | 20–30 字 | 自我定位 + 岗位角色 + 独特点 | "在校期间独立运营 AI × 金融信息方向内容账号 X @Cady_btc" |
| 2. 核心数据 | 15–30 字 | 最能打的 2–3 个数字 | "零投放做到 **9,000+** 粉、**1,300+** 订阅、**20+** 次品牌商单" |
| 3. 工具栈 | 15–25 字 | AI 工具矩阵（JD 提到的排前） | "日常用 Claude Code、Kimi、ChatGPT、Cursor 做内容生产与小工具开发" |
| 4. 经历覆盖面 | 20–30 字 | 实习 / 项目抽象概括 | "两段运营实习覆盖海外 AIGC 冷启动、竞品拆解、活动数据复盘与跨部门协作" |
| 5. 能力锚点 | 15–30 字 | 专业 / 语言 / 软素质 | "英语 CET-6。应用统计出身，习惯用数据面板复盘每条内容的真实表现" |

### 5.2 结合岗位的槽位调整

**读 JD → 提取 Top 3 关键词 → 改写槽位**：

| JD 强调 | 槽位调整 |
|---|---|
| 海外市场 / 英语 | 槽位 1 加"海外 / 出海"定位；槽位 5 加"英语 CET-X，可独立撰写英文内容" |
| 社媒 KOL / 内容增长 | 槽位 2 把粉丝数和商单数前置；槽位 4 加"达人共创 / 爆款冷启动" |
| AI 产品 / Prompt 工程 | 槽位 3 工具栈细化（Claude / Cursor / 具体 AI 生产力工具）；槽位 4 加"vibe coding 作品" |
| 数据分析 / 增长 | 槽位 5 把应用统计专业前置；槽位 4 强调"AB 测试 / 归因模型" |
| 金融科技 | 槽位 1 保留"AI × 金融信息"；槽位 4 加"金融消费场景"（如杭银经历）|

### 5.3 写作红线

- ❌ **不要写目标和抱负**："希望加入 XX 公司，实现自我价值"
- ❌ **不要写性格标签**："ENFP / 开朗活泼 / 团队合作者"（除非 JD 明确求这个）
- ❌ **不要写学分绩 / 排名**（除非 Top 5%，否则反效果）
- ✅ 所有数据必须在下文的项目/实习里能对应找到（不能个人总结写了下文却没有）

## 六、数字写作规则

### 6.1 字面保留（硬约束）

| ✅ 正确 | ❌ 错误 | 原因 |
|---|---|---|
| 9,000+ | 9k+ / 9K / 9千 | 原文数字字面必须保留 |
| 10,000+ | 10k+ / 1w+ | 除非原文就是 "1w+" |
| 1,048 万 → 1,262 万 | 1048万 → 1262万 | 千分位逗号不能丢 |
| **18.9%** | 18.9 % / **18.9**% | 百分号紧贴数字 |
| **+20.6%** | 20.6%（丢 +号） | 增长符号必须保留 |

### 6.2 组合写法

| 场景 | 写法 |
|---|---|
| 单数据增长 | **18.9%** |
| 单场/单月加数据 | 单场参与率提升 **18.9%** |
| 前后对比 + 百分比 | **1,048 万 → 1,262 万（+20.6%）** |
| 两个独立百分比 | 从 **4%** 提升到 **10%** |
| 数字区间 | 每日拉 50+ 条资讯 / 筛 5-10 条素材（过程描述不加粗） |

### 6.3 数字放在哪里？

- **个人总结**：只放 2–3 个最硬的数字，不超过 3 个
- **bullet 里**：每条 bullet 至少 1 个数字（没数字的 bullet 是水 bullet）
- **开头还是结尾**：
  - 动作动词开头，结果数字结尾（≈ 80% 情况）
  - 偶尔把核心数字前置（强调结果）——不超过 20%

## 七、专业技能写作模板

### 7.1 分类层级

**标准三分法**（参考 layout rules 第六节）：

```
• **AI 工具与内容生产：** [AI 工具 1–7 个]；[设计工具 1–3 个]
• **海内外社媒：** [深度运营平台]；[深度使用平台]         ← 可删
• **数据、开发与语言：** [技术栈 5–8 个]；英语 CET-X、计算机 X 级
```

### 7.2 工具写作规则

| 规则 | 说明 |
|---|---|
| 按 JD 权重排序 | JD 提到 Kimi → Kimi 排 AI 工具首位 |
| 不列默认工具 | Word / Excel / PPT / Email 不写 |
| 加熟练度信号 | 「深度运营」「日常重度用户」「个人模板库 60+」 |
| 工具名大小写准确 | ChatGPT（不是 chatgpt / ChatGpt） / NotebookLM / MySQL |
| 顿号「、」分主类 | 同类内部顿号，跨类用分号「；」 |

### 7.3 不要做的事

- ❌ **不要造技能等级**："精通 / 熟练 / 了解" 这些字本身模糊
- ❌ **不要写太长列表**：每行超过 80 字考虑拆分或删非核心
- ❌ **不要放软素质**："沟通 / 领导力 / 责任心" → 这些放到个人总结或实习 bullet

## 八、经历排序与取舍

### 8.1 项目 / 实习内部排序（与时间无关）

**项目经历**按**与 JD 相关性**排序：
- 高相关 + 高数据 → 第一位
- 开源作品 / vibe coding → 第二位（作为"作品集锚点"）
- 其他老项目 → 后面或删掉

**实习经历**按**时间倒序**排（近期的放上）。

### 8.2 什么时候删一条 bullet？

**删的信号**：
- 没数字 → 大概率是水 bullet
- 和 JD 关键词无任何交集 → 占位
- 重复了另一条 bullet 的信息 → 合并
- 超过 90 字且说不清 → 写糊了

**保留的信号**：
- 量化数据硬
- 命中 JD 2+ 关键词
- 体现 0→1 或大幅提升
- 是"稀缺能力"（如自运营账号、vibe coding）

### 8.3 什么时候删一整个项目？

- 和目标 JD 方向完全无关（例：学生会活动 vs AI 运营岗）
- 没有任何量化结果可提炼
- 时间太早且质量不高（大一做的课程作业）

## 九、岗位匹配优化（投递前必检 10 步）

> 本节是工作流版本，详细规则看 `resume_layout_rules.md` 第十节。

1. 读 JD，标出 **Top 5 关键词**（按出现频次 + 任职要求权重）
2. 个人总结**第一句必须带 Top 1 关键词**
3. 求职意向行改为 **JD 原岗位名**（不是"AI 运营"这种泛化）
4. bullet label 优先用 **JD 里的措辞**（例：JD 说"跨团队协作" → label 用"跨团队协作"，不用"跨部门"）
5. 专业技能里 **JD 提到的工具排最前**
6. 如果 JD 特别强调某项能力（如"英语流利"、"视频制作"），**个人总结 + 技能区双重强化**
7. **强相关的项目上移**到第一位（即便时间上不是最新）
8. **弱相关的 bullet 删掉或改写**
9. 头部到岗时间备注 **对应 JD 要求**（"每周 5 天"/"实习 3+ 月"）
10. 生成后**人眼扫一遍 5 秒测试**：HR 看头 3 行能否 100% 确定你在投这个岗？

## 十、常见错误与改写示范

### 10.1 错误 1：动词弱，结果无

❌ "参与公司海外社媒账号的运营和维护工作"
✅ "**海外社媒 0 → 1：** 独立搭建 TikTok + Instagram 官方账号，**3 个月** 做到单条 **10,000+** 自然曝光"

### 10.2 错误 2：形容词堆砌

❌ "成功显著提升了活动参与率，效果非常出色"
✅ "**活动数据复盘：** 搭建活动日报 + 归因模型，单场参与率提升 **18.9%**（1,240 → 1,644 人）"

### 10.3 错误 3：虚词开头

❌ "负责 / 参与 / 协助 / 主要做"
✅ 直接用具体动词：搭 / 写 / 跑 / 做 / 拆 / 带 / 调 / 主导

### 10.4 错误 4：长句 + 叠罗汉

❌ "负责利用各种工具和资源进行用户增长工作，通过各种方式驱动业务发展，助力团队实现增长目标"
✅ "**用户增长：** 搭建 X API + Claude 内容管道，每周稳定产出 15+ 条内容；粉丝 **0 → 9,000+**"

### 10.5 错误 5：数据拆分错位

❌ "粉丝增长到 9000 人，订阅 1300 人，商单 20 次"（没单位，没加粗）
✅ "X 粉丝 **0 → 9,000+**，Telegram 订阅 **0 → 1,300+**，完成 **20+** 次品牌商单"
