# 简历自动生成 · 加粗规则（Bold Rules）

> **适用范围**：CareerOS 所有简历自动生成场景（master_resume.yaml → AI tailor → 定制简历）
> **最后更新**：2026-04-20
> **维护原则**：规则变更必须同步更新本文件 + `resume_system.md`

---

## 一、必须加粗（标题类）

### 1.1 结构性标题

- 候选人**姓名**（楷体 / Kaiti SC，字号 36pt，weight 500+）
- **Section 标题**：个人总结 / 项目经历 / 实习经历 / 专业技能 / 教育背景（灰底 + 黑体 + 加粗）
- **三段式 header**：公司名 / 职位 / 日期（10pt + 加粗）

### 1.2 Bullet label（冒号前的短语）

每条 bullet 首部的标签一律加粗，例如：

- **内容生产线：** …
- **Prompt 工程与增长变现：** …
- **达人共创：** …
- **产品定义：** …
- **技术实现：** …
- **开发方法论：** …
- **海外社媒 0 到 1：** …
- **自然流量破圈：** …
- **竞品调研与付费冷启动：** …
- **活动数据复盘：** …
- **跨部门触达机制：** …
- **内容冷启动：** …

### 1.3 技能与教育分类 label

- **AI 工具：** …
- **内容生产：** …
- **数据与开发：** …
- **语言与证书：** …
- **核心课程：** …
- **荣誉：** …

## 二、量化数据：只加粗纯数字，不加粗单位与描述

**核心原则**：`<strong>` 标签只包住纯数字（含 `+` `%` `万` `w` `→` `-` 等数字连接符），不包住前后的单位、名词、动词。

### 2.1 标准对照表

| 原文 | 正确加粗写法 |
|---|---|
| 9,000+ 粉 | `**9,000+** 粉` |
| 1,300+ 订阅 | `**1,300+** 订阅` |
| 20+ 次品牌商单 | `**20+** 次品牌商单` |
| 60+ 条个人 Prompt 模板 | `**60+** 条个人 Prompt 模板` |
| 15+ 条（内容） | `**15+** 条` |
| 200+ 粉增长 | `**200+** 粉增长` |
| 100+ 国内外 AI 公司 | `**100+** 国内外 AI 公司` |
| 10,000+ 自然曝光 | `**10,000+** 自然曝光` |
| 30+ 页竞品画像 | `**30**+ 页竞品画像`（只粗 30） |
| 2w+ 播放 | `**2w+** 播放` |
| 单场活动参与率提升 18.9% | 单场活动参与率提升 `**18.9%**` |
| 单场触达率从 4% 提升到 10% | 单场触达率从 `**4%**` 提升到 `**10%**` |
| 成本下降 40% | 成本下降 `**40%**` |
| UV 从个位数提升到 200+ | UV 从个位数提升到 `**200+**` |

### 2.2 数字之间的连接符（整段加粗）

当数字构成"前 → 后"或"起 → 止"的对比，整段（含单位）加粗：

| 原文 | 加粗写法 |
|---|---|
| 0 → 9,000+ | `**0 → 9,000+**` |
| 0 → 1,300+ | `**0 → 1,300+**` |
| 1,240 → 1,644 | `**1,240 → 1,644**` |
| 1,048 万 → 1,262 万（+20.6%） | `**1,048 万 → 1,262 万（+20.6%）**` |

## 三、整句不加粗（过程描述）

即便句子里有数字，若属于**过程描述 / 规格参数 / 流程说明**而非**结果展示**，整句不加粗：

| 原文 | 加粗处理 |
|---|---|
| 单条内容生产时间从 40 分钟压到 10 分钟以内 | **整句不加粗** |
| 单条 JD 定制简历 30 秒产出 | **整句不加粗** |
| 每日自动拉 50+ 条资讯、Prompt 模板筛选 5-10 条素材 | **整句不加粗** |

### 判断标准

> **这个数字能不能写进复盘报告的「outcome」一栏？**
> - 能 → 加粗（增长率、转化率、绝对规模、商业变现数）
> - 不能 → 不加粗（处理速度、配置参数、流程中间量）

---

## 四、负面清单（禁止加粗）

以下一律不加粗，即便看起来重要：
- 定性描述词（「首批付费用户」「商业价值验证」「头部竞品」）
- 动词短语（「搭建」「重做」「深度拆解」）
- 工具 / 平台名称（Claude Code、Kimi、TikTok——这些不是数据）
- 公司名、职位名（这些由 header 样式控制，不靠内联 strong）

---

## 五、实施自检清单（每次生成后过一遍）

- [ ] 所有 section 标题、bullet label 都是粗体
- [ ] 所有 outcome 型数据（增长、转化、规模、收入）的**纯数字**是粗体，单位不是
- [ ] 过程描述型数字未被加粗
- [ ] 没有把动词、连词、工具名误加粗
- [ ] 一页 A4 容纳，无越页

## 六、HTML 实现约定

### 6.1 CSS

```css
b, strong { font-weight: 700; }
.section-title { font-weight: 700; }   /* 灰底标题 */
.exp-head td { font-weight: 700; }     /* 三段式 header */
.skill-label { font-weight: 700; }     /* 技能分类 label */
```

### 6.2 Bullet 写法模板

```html
<div class="bi"><strong>Bullet Label：</strong>正文描述，量化数据用 <strong>数字</strong> 单位 的形式</div>
```

### 6.3 示例（正确）

```html
<div class="bi"><strong>自然流量破圈：</strong>零投放条件下做到单条内容 <strong>10,000+</strong> 自然曝光，带动官网日均 UV 从个位数提升到 <strong>200+</strong></div>
```

### 6.4 示例（错误）

```html
<!-- ❌ 错误 1：把单位也加粗 -->
<div class="bi">做到<strong>单条内容 10,000+ 自然曝光</strong></div>

<!-- ❌ 错误 2：把过程描述加粗 -->
<div class="bi"><strong>单条 JD 定制简历 30 秒产出</strong></div>

<!-- ❌ 错误 3：把动词加粗 -->
<div class="bi">基于 <strong>搭建</strong>信息抓取管道</div>
```
